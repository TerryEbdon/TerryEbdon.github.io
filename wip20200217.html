RSTS/E Host Comms

Thoughts about connecting RSTS/E jobs to an app running on the host machine. One possibility is to model the system as a remote job entry terminal, on the host, that submits RSTS/E batch jobs and gets notified when they complete. The tricky part is moving files between the systems, while avoiding issues caused by the PDP-11 emulator locking files attached to virtual hardware. A work-around could be the use of .do files (emulator scripts) that attach / detach the files based on console messages generated by a RSTS/E job, e.g. a BASIC-PLUS program.
